# 지도 UI 전면 개편 스펙

**날짜**: 2026-03-24
**범위**: 지도 탭 (`MapView` 및 관련 컴포넌트)
**레퍼런스**: 네이버 부동산 (3-panel 데스크탑), 직방 (모바일 하단 시트)

---

## 1. 목표

현재 지도 탭은 핀 클릭 시 팝업이 뜨고, 목록과 지도가 시각적으로 단절되어 있다.
개편 후에는 지도·목록·상세가 항상 연동되는 통합 뷰로 만든다.

---

## 2. 레이아웃

### 데스크탑 (≥ 768px)

```
┌─────────────────────────────────────────────────────┐
│  topbar (기존 유지)                                    │
├──────────┬────────────────────────┬──────────────────┤
│  좌측 패널 │       지도 (카카오맵)    │   우측 상세 패널   │
│  280px   │       flex: 1         │   380px          │
│  고정     │       고정             │   선택시 슬라이드인  │
│          │                       │                  │
│ [필터]    │  가격 핀 마커           │  이미지 갤러리      │
│ [목록]    │  선택 핀 강조           │  가격/면적/층수     │
│  스크롤   │                       │  태그 칩           │
│          │                       │  원본 보기 버튼     │
└──────────┴────────────────────────┴──────────────────┘
```

- **좌측 패널 (280px, 고정)**
  - 상단: 필터 영역 (플랫폼 칩, 월세 범위, 면적, 층, 사진 여부)
  - 하단: 현재 지도 범위 내 매물 목록 (스크롤), 건수 표시
  - 카드 hover → 해당 핀 강조
  - 카드 클릭 → 지도 pan + 우측 상세 패널 오픈

- **중앙 지도 (flex: 1)**
  - 카카오맵 풀사이즈
  - 핀: 가격 라벨 표시 (`45만`, `60만`), 선택시 다크 강조
  - 지도 이동/줌 → 좌측 목록 자동 갱신
  - InfoWindow 팝업 제거 (→ 우측 패널로 대체)

- **우측 상세 패널 (380px)**
  - 기본: 숨김 (지도가 공간 차지)
  - 핀 또는 목록 카드 클릭 시 슬라이드인
  - 내용: 이미지 갤러리, 가격 헤더, 상세 정보 그리드, 원본 보기
  - ✕ 닫기 → 패널 닫힘, 핀 선택 해제

### 모바일 (< 768px)

```
┌──────────────────────┐
│  상단 필터 바 (고정)    │
├──────────────────────┤
│                      │
│   지도 (전체 화면)     │
│                      │
│  ┌────────────────┐  │
│  │ ↑ 목록 (58건)  │  │  ← 하단 시트
│  │ 드래그업 → 전체 │  │
│  └────────────────┘  │
└──────────────────────┘
```

- 상단: 필터 칩 가로 스크롤 바
- 지도 전체 화면
- 하단 시트: peek(64px) → 절반 → 전체 3단계 드래그
- 핀 클릭 → 하단 시트 peek에 카드 1개 표시
- 시트 올리면 목록 전체 표시

## 3. 컴포넌트 변경

### 신규
| 파일 | 역할 |
|------|------|
| `MapLeftPanel.jsx` | 필터 + 목록 (좌측 패널) |
| `MapRightPanel.jsx` | 매물 상세 (우측 패널, 슬라이드) |
| `MapBottomSheet.jsx` | 모바일 하단 시트 |

### 수정
| 파일 | 변경 내용 |
|------|-----------|
| `MapView.jsx` | 3-panel 레이아웃 구조로 재작성, 모바일/데스크탑 분기 |
| `MapSidePanel.jsx` | → `MapLeftPanel` + `MapRightPanel`으로 분리 (기존 제거) |
| `MapFilters.jsx` | `MapLeftPanel` 내부로 이동 |
| `KakaoMap.jsx` | InfoWindow 팝업 제거, 핀 클릭 콜백만 유지 |
| `styles.css` | 3-panel CSS, 하단 시트 CSS 추가 |

## 4. 인터랙션 흐름

```
핀 클릭
  → selectedId 업데이트
  → 좌측 목록: 해당 카드로 스크롤 + 강조
  → 우측 패널: 슬라이드인 + 상세 fetch

목록 카드 클릭
  → 지도: 해당 핀으로 pan + zoom
  → 핀 강조
  → 우측 패널: 슬라이드인

필터 변경
  → 지도 마커 새로고침
  → 좌측 목록 새로고침

우측 패널 닫기(✕)
  → 패널 닫힘
  → selectedId null
  → 핀 강조 해제
```

---

## 5. 상태 관리

`MapView`가 단일 진실의 원천으로 유지:

```js
selectedId    // 선택된 매물 ID (핀 강조 + 패널 표시)
filters       // 필터 상태
bounds        // 현재 지도 범위
rightPanelOpen // 우측 패널 열림 여부
```

## 7. 성공 기준

- [ ] 핀 클릭 시 팝업 없이 우측 패널 슬라이드인
- [ ] 좌측 목록과 지도 핀 양방향 연동 (클릭/강조 공유)
- [ ] 필터 변경 시 지도+목록 동시 갱신
- [ ] 모바일에서 하단 시트 드래그 동작
- [ ] 기존 즐겨찾기, 상세 조회 기능 그대로 유지
