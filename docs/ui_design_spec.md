# 서울 원룸/투룸 통합 매물 서비스 UI 디자인 스펙 v2

## 0) 최종 선택 톤 (2026 트렌드)
- 선택 톤: **Liquid Glass 2.0 (투명-입체형 신뢰 인터페이스)**
- 명칭: `RF Glass Trust`
- 핵심 컨셉: **데이터 신뢰성 + 부드러운 유리 레이어 + 짙은 정보 밀도의 시각적 정돈**
- 한 줄 정의: "투명한 층위를 쌓아 정보 우선순위를 드러내고, 가격·면적·신뢰도는 가장 강하게 판독되게 하는 탐색형 대시보드"
- 왜 이 톤인지
  - 2026 플랫폼 UI 흐름(글래스/입체/유동 레이어)은 주류 플랫폼에서 채택 속도가 매우 빠름
  - 데이터 중심 서비스에서 `가독성`, `주의집중`, `상태 구분`을 동시에 처리하기 쉬움
  - 경쟁 플랫폼 대비 “고급스러움 + 신뢰감” 인상을 동시에 줄 수 있음

## 1) 비주얼 원칙
- 탐색 속도 우선: 첫 화면에서 핵심 수치가 2초 안에 보인다.
- 계층 가독성: 배경-콘텐츠-메타어트리뷰트의 3층 레이어만 유지.
- 신호 최소화: 경고는 빨간색 과다 사용 지양, warning chip + 아이콘 + 소제목으로 한 번에 처리.
- 정보 과열 방지: 텍스트 과다 노출 대신 컴팩트 카드 + 확장 패널.

## 5) 핵심 컴포넌트 표준

### 5.1 헤더
- 높이: `72px`
- 배경: 글래스 패널 + 강한 블러
- 스크롤 시: 더 어두운 반투명 배경 + 얇은 테두리

### 5.2 검색 패널
- 형태: 반투명 패널 `rf-glass-panel`
- 패딩: `18px`
- 반올림: `20px`
- 입력 필드 높이: `46px`
- 필드 배경: `rgba(255,255,255,0.5)`
- 포커스: 테두리 `rgba(59,130,246,0.5)` + 미세 글로우

### 5.3 카드
- 카드 radius: `22px`
- 패널: `backdrop-filter blur(18px)` + 투명도 0.8 이하
- 그림자: `0 14px 32px rgba(1, 10, 34, 0.32)`
- 하단 CTA 영역 고정 높이: `44px`

### 5.4 배지
- 크기: `height 26` min `86px`
- 내부 여백: `4px 12px`
- 아이콘 + 텍스트 1줄 고정
- 상태별 외곽선 강화

### 5.5 버튼
- Primary: `height 44~48`, radius `12`, 글래스 보더 1px
- Secondary: 투명 바탕 + 얇은 라인
- Icon-only: 정사각형 44, 라운딩 12
- 로딩: 버튼 내부 shimmer

### 5.6 모달/드로어/툴팁
- 모달 배경: `rgba(3, 8, 24, 0.65)`
- 컨텐츠 패널: 글래스 `radius 20`
- 팝오버는 마우스/터치 근접 오프셋 10px

## 6) 화면별 구현 지시 (v2)

### 6.1 통합 검색 화면
- 레이아웃: 상단 글래스 검색 바 + 빠른 조건 토글 + 하단 고정 액션 바
- 검색 실행 버튼은 2단 컬러 블록: `primary + subtle`
- 최근 필터는 칩으로 상단에 누적 노출
- 검색 조건 누락시 CTA 회색 비활성 + inline 이유

### 6.2 매물 리스트 화면
- 목록은 `RF Listing Card`만 사용, 텍스트 밀도는 낮추되 숫자 대비는 높임
- 가격/보증금 숫자 강조는 `title + body-lg`
- 신뢰도/오탐/상태 배지는 상단 좌우 배치
- 썸네일 영역은 `aspect-ratio: 16/10` + 오버레이 가격칩

### 6.3 매물 상세 화면
- 3단 구성: 핵심 히어로 / 정규화 상세 / 매칭 패널
- 정규화/원문 전환은 탭(글래스 탭)
- 경고 섹션은 경고 라벨 + 경로(`원문값`, `추정값`, `재확인 필요`) 3컬러 설명
- 원본 링크는 하단 고정 버튼 그룹에서 우선 노출

### 6.4 중복 검토 화면
- 각 그룹을 “글래스 토글 패널”로 구성, 펼침/접힘 애니메이션 사용
- 후보군 비교는 `before/after 스타일`이 아니라 좌우 3분할 카드로 정렬해 상대 비교를 직관화
- 2차 판단 액션은 `동일군`/`분리`/`보류` 3개 버튼 한 줄

### 6.5 품질/수집 상태 대시보드
- 탭: `플랫폼`, `파이프라인`, `품질 경보`
- 플랫폼 카드에 실패율/상태/최근 오류 1줄 표시
- 수치 차이는 게이지형 + 텍스트 동반(예: `3.4%▲ +0.8%`)

### 6.6 마이 페이지
- 알림 조건은 카드형 블록, 각 블록은 글래스 패널
- 신고 내역은 상태별 컬러 토큰 맵핑(접수/검토/완료)

## 7) 상호작용·애니메이션
- 기본 트랜지션: `160ms cubic-bezier(.2,.8,.2,1)`
- 카드 입장: `y 10px -> 0`, `opacity 0 -> 1`
- 패널 전환: `80ms` fade + `20px` 이동
- 필터 열기/닫기: 1개씩 순차 등장 (stagger 40ms)
- 과도한 파티클/3D 모션 금지

## 8) 상태/빈 상태/오류
- 로딩: glass shimmer + 예상 소요 시간 1줄
- 실패: 단순 에러가 아니라 “재시도 경로 + 원인 간단 설명” 제공
- 빈 목록: 조건 완화 제안 chip 4개 자동 제시
- 경고(추정치): 카드 내에서 붉은색은 텍스트 1회 + 아이콘 1회만 사용(오버로드 금지)

## 9) 접근성
- 최소 터치: `48px`
- 폰트 대비: 본문 대비 4.5:1 확보 기준
- 포커스 아웃라인: `2px solid rgba(103, 232, 249, 0.85)`
- 다크/저채도 모드 전환용 대비 토큰 추가 예정(2차 스펙)
